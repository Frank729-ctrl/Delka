API Reference

Base URL: https://api.delkaai.com

Authentication

All /v1/* endpoints require the X-DelkaAI-Key header.

X-DelkaAI-Key: fd-delka-sk-yoursecretkey

Use a Secret Key (sk-) for server-to-server calls. Publishable Keys (pk-) have restricted access.

Health Check

GET /v1/health Public

Returns API status. No authentication required.

curl https://api.delkaai.com/v1/health
{"status": "ok", "version": "1.0.0"}

CV Generation

POST /v1/cv/generate SK required

Generate a professional CV as a PDF. Returns PDF bytes or a job_id if a webhook is provided.

{
  "full_name": "Jane Smith",
  "email": "jane@example.com",
  "phone": "+1-555-0100",
  "location": "New York, NY",
  "summary": "Senior software engineer with 8 years experience.",
  "experience": [
    {
      "company": "Acme Corp",
      "title": "Senior Engineer",
      "start_date": "2020-01",
      "end_date": "present",
      "bullets": ["Led microservices migration", "Reduced latency by 40%"]
    }
  ],
  "education": [
    {"school": "MIT", "degree": "BSc Computer Science", "year": "2016"}
  ],
  "skills": ["Python", "FastAPI", "PostgreSQL"],

  "template": "bold_header",
  "color": "professional_blue"
}

Webhook Payload

When a job completes, DelkaAI POSTs to your webhook URL:

{
  "job_id": "uuid",
  "status": "complete",
  "event": "cv.generated",
  "data": {
    "pdf_base64": "JVBERi0x...",
    "template": "bold_header",
    "color": "professional_blue"
  },
  "timestamp": "2025-01-01T12:00:00"
}

If HMAC is enabled, verify the X-DelkaAI-Signature header against your HMAC secret.
